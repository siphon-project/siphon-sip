# Commercial support

SIPhon is MIT-licensed and free to run in production — no open-core holdbacks, no paid
tier of the protocol engine. If you'd rather not carry the integration and operations
alone, commercial support is available from
**[Real Time Telecom B.V.](https://realtime-telecom.nl)**, run by SIPhon's maintainer.

## What RTT can help with

- **Deployment & topology design** — single node, a redundant pair / N nodes, or a full
  IMS core (P-CSCF / I-CSCF / S-CSCF). See [Deployment & operations](deployment.md) and
  [Scaling & redundancy](scaling-and-redundancy.md) for the shape this builds on.
- **IMS / VoLTE / VoNR integration** — Cx / Rx / Sh / Rf Diameter, IPsec sec-agree, AKA,
  iFC / ISC routing, and 5G N5 / Nbsf — wired against your HSS, PCF, and CSCFs.
- **Migration from Kamailio / OpenSIPS** — porting routes and `clusterer` / `dmq_usrloc`
  topologies. See [Migrating from Kamailio / OpenSIPS](migrating-from-kamailio-opensips.md).
- **Custom scripting & feature development** — Python handlers built to your call flow,
  upstreamed into the project where it makes sense.
- **Performance tuning** — profiling and capacity planning against your real traffic mix.
- **SLA-backed support** — production response commitments.

[Get in touch via realtime-telecom.nl →](https://realtime-telecom.nl)

## Sponsor the project

Want a particular feature built or fast-tracked? Feature sponsorship funds work that
lands in the open-source project — your use case ships sooner, and everyone downstream
benefits. Use the **Sponsor** button on the
[GitHub repository](https://github.com/siphon-project/siphon-sip), or reach out through
RTT to scope it.

Ongoing development is backed by
**[Real Time Telecom B.V.](https://realtime-telecom.nl)**, the project's founding sponsor.
